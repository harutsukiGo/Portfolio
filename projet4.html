QueueApp

Lors de mon stage de deuxième année, j'ai développé une application de gestion de file d'attente.

En quoi le développement d’une application de gestion de file d’attente répond à un besoin crucial d’organisation ?

DNA Webhosting, une entreprise malgache spécialisée dans le développement de solutions technologiques, m'a confié le développement d’une application de gestion de file d'attente. Mon rôle consistait à concevoir, conceptualiser et développer l’application. J’ai utilisé Docker pour déployer mes applications, PHP pour le backend et JavaScript pour le frontend. J’ai également utilisé une base de données MySQL et le service de websockets de Pusher.
Pour le backend, j’ai choisi d’utiliser principalement PHP avec une architecture MVC (Modèle-Vue-Contrôleur). Cette architecture permet d’avoir un code organisé, maintenable, avec des séparations claires de chaque rôle et évolutive.
Pour le frontend, j’ai choisi HTML/CSS et JavaScript pour ajouter du dynamisme.
Pour améliorer l’expérience utilisateur en évitant les rechargements de page pour voir les données actualisées, j’ai opté pour Pusher, un service en ligne qui simplifie la gestion des communications en temps réel via les WebSockets. Le principe est simple : les clients peuvent s’abonner à des canaux pour recevoir instantanément des événements. J’ai donc créé un canal pour chaque objet de ma base de données. Lorsqu’un événement se produit dans l’application (par exemple, la création d’un nouvel agent), le serveur envoie une requête à l’API de Pusher pour publier cet événement sur le canal correspondant. Pusher transmet alors automatiquement l’événement à tous les clients abonnés, établissant une connexion persistante entre les clients et ses serveurs, garantissant ainsi une diffusion en temps réel.

J’ai pu développer mes compétences en développement web avec du PHP.
J'ai découvert l'utilisation des WebSockets pour le temps réel avec Pusher.
