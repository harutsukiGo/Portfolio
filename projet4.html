Description du projet

DNA Webhosting, une entreprise malgache spécialisée dans le développement de solutions technologiques, m'a confié le développement d'une application de gestion de file d'attente. Mon rôle consistait à concevoir, conceptualiser et développer l'application.

Architecture MVC

Code organisé, maintenable avec des séparations claires de chaque rôle.

Temps réel

WebSockets via Pusher pour des mises à jour instantanées sans rechargement.

Déploiement Docker

Conteneurisation de l'application pour un déploiement simplifié.

Interface dynamique

Frontend en HTML/CSS et JavaScript pour une expérience utilisateur fluide.

Réalisation technique

Pour le backend, j'ai choisi d'utiliser principalement PHP avec une architecture MVC (Modèle-Vue-Contrôleur). Cette architecture permet d'avoir un code organisé, maintenable, avec des séparations claires de chaque rôle et évolutive. J'ai utilisé Docker pour déployer mes applications et une base de données MySQL.

Pour le frontend, j'ai choisi HTML/CSS et JavaScript pour ajouter du dynamisme.

Pour améliorer l'expérience utilisateur en évitant les rechargements de page pour voir les données actualisées, j'ai opté pour Pusher, un service en ligne qui simplifie la gestion des communications en temps réel via les WebSockets. Le principe est simple : les clients peuvent s'abonner à des canaux pour recevoir instantanément des événements. J'ai donc créé un canal pour chaque objet de ma base de données. Lorsqu'un événement se produit dans l'application (par exemple, la création d'un nouvel agent), le serveur envoie une requête à l'API de Pusher pour publier cet événement sur le canal correspondant. Pusher transmet alors automatiquement l'événement à tous les clients abonnés, établissant une connexion persistante entre les clients et ses serveurs, garantissant ainsi une diffusion en temps réel.
